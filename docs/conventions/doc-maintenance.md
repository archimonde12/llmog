# Documentation Maintenance Guidelines

## Definition of Sufficient Documentation
- A new developer can identify key services and run the basic workflow.
- On-call can quickly find the correct runbook for common incidents.
- A reviewer can look up approved technical decisions.

## Required Metadata
Applies to:
- `docs/decisions/*.md` (excluding README/template files)
- `docs/runbooks/**/*.md`
- `docs/services/*.md`
- primary convention documents in `docs/conventions/`
